# AI学习教育系统 - 小白项目说明文档

## 1️⃣ 项目是做什么的

这是一个**AI学习教育系统**，简单来说就是一个专门用于学习的网站，它可以根据不同用户角色（学生、教师、管理员）提供不同的功能。

**举个例子**：
- 如果你是学生，登录后可以看到「AI助教」「学习进度」「学习工具」等功能
- 如果你是教师，登录后可以看到「数据分析」「作业管理」「课程管理」等功能
- 如果你是管理员，登录后可以管理整个系统的用户和权限

**项目亮点**：PROJECT_HIGHLIGHTS.md文档

## 3️⃣ 项目目录结构说明

项目的目录结构就像一个文件夹树，每个文件夹都有不同的用途：

```
AI-web-master/          # 项目根目录
├── public/             # 存放静态资源（如网站图标）
├── src/                # 前端的主要代码目录
│   ├── assets/         # 图片、SVG图标等资源
│   ├── components/     # 可复用的界面组件
│   ├── layouts/        # 网站的布局模板
│   ├── locales/        # 多语言配置文件
│   ├── router/         # 页面路由配置
│   ├── service/        # 与后端通信的服务
│   ├── store/          # 数据状态管理
│   ├── views/          # 网站的各个页面
│   ├── App.vue         # 应用的根组件
│   └── main.ts         # 应用的入口文件
├── backend/            # 后端代码目录
│   ├── src/            # 后端源代码
│   ├── prisma/         # Prisma ORM配置
│   └── package.json    # 后端项目配置
├── package.json        # 前端项目配置文件
└── README.md           # 项目说明文档
```

### 关键目录详解

- **src/views/**：存放网站的所有页面，比如学生页、教师页、AI助教页等
- **src/components/**：存放可复用的界面元素，比如按钮、菜单、表格等
- **src/router/**：管理页面之间的跳转，比如从登录页跳转到首页
- **src/layouts/**：定义网站的整体布局，比如顶部导航栏、左侧菜单等
- **src/service/**：处理与后端服务器的通信，获取和发送数据
- **backend/src/**：存放后端API和业务逻辑代码
- **backend/prisma/**：存放数据库模型和迁移文件

## 4️⃣ 项目运行流程

当你打开这个网站时，发生了以下过程：

1. **加载入口文件**：浏览器首先加载 `src/main.ts` 文件，这是整个应用的入口
2. **初始化应用**：创建Vue实例，设置路由、状态管理等
3. **渲染根组件**：将 `App.vue` 渲染到页面上
4. **加载布局**：根据当前页面选择合适的布局模板
5. **渲染页面**：加载并渲染当前URL对应的页面内容
6. **获取数据**：如果页面需要数据，会通过 `service` 模块从后端获取
7. **用户交互**：用户可以与页面交互，触发各种功能

## 5️⃣ 如何在本地部署并运行

### 准备工作

在开始之前，你需要安装以下软件：

1. **Node.js**：用于运行JavaScript代码，下载地址：https://nodejs.org/（选择LTS版本）
2. **VS Code**：用于查看和编辑代码，下载地址：https://code.visualstudio.com/
3. **PostgreSQL**：用于存储数据，下载地址：https://www.postgresql.org/download/

### 详细步骤

#### 第一步：安装依赖

打开命令行工具（Windows系统使用PowerShell或CMD），执行以下命令：

```cmd
# 进入项目目录
cd d:\AI-web-master

# 安装前端依赖
pnpm install

# 安装后端依赖
cd backend
pnpm install
```

**为什么要运行这个命令？**：项目需要很多依赖包才能运行，这个命令会自动下载并安装所有必需的包。

#### 第二步：配置数据库

1. 启动PostgreSQL数据库服务
2. 创建一个新的数据库
3. 在 `backend/.env` 文件中配置数据库连接信息

#### 第三步：启动后端服务

```cmd
# 在backend目录下执行
pnpm start
```

**为什么要运行这个命令？**：这个命令会启动后端服务器，提供API接口。

#### 第四步：启动前端服务

打开另一个命令行窗口，执行以下命令：

```cmd
# 在项目根目录下执行
pnpm dev
```

**为什么要运行这个命令？**：这个命令会启动前端开发服务器，让你可以在浏览器中查看和测试网站。

#### 第五步：访问网站

命令执行成功后，你会看到类似这样的输出：

**后端服务**：
```

[Nest] 12345  - 04/17/2026, 3:30:01 PM     LOG [HttpAdapterHost] Nest application is running on: http://localhost:3001
```

**前端服务**：

复制 `http://localhost:9527/` 到浏览器中访问，就可以看到网站了！

### 常见错误及解决方法

1. **错误**：`pnpm command not found`
   **解决方法**：先安装pnpm：`npm install -g pnpm`

2. **错误**：安装依赖时网络超时
   **解决方法**：检查网络连接，或使用国内镜像：`pnpm config set registry https://registry.npmmirror.com`

3. **错误**：启动服务器时端口被占用
   **解决方法**：使用其他端口：
   - 前端：`pnpm dev --port 9528`
   - 后端：修改 `backend/src/main.ts` 中的端口配置

4. **错误**：数据库连接失败
   **解决方法**：检查PostgreSQL服务是否启动，数据库连接信息是否正确

## 6️⃣ 项目中最重要的3～5个文件讲解

### 1. `src/main.ts` - 前端应用的入口文件

**作用**：这是前端项目的「大脑」，负责初始化整个应用。

**主要功能**：
- 导入并配置Vue实例
- 设置路由和状态管理
- 注册全局组件和插件
- 将应用挂载到浏览器中

### 2. `src/App.vue` - 前端应用的根组件

**作用**：这是前端项目的「骨架」，所有页面都会嵌套在这个组件中。

**主要功能**：
- 定义应用的基本结构
- 处理全局的布局和样式
- 包含网站的主要容器

### 3. `src/router/index.ts` - 前端路由配置文件

**作用**：这是前端项目的「导航员」，负责管理页面之间的跳转。

**主要功能**：
- 定义所有页面的URL路径
- 配置页面的访问权限
- 处理页面跳转逻辑

### 4. `src/service/request/index.ts` - 前端网络请求文件

**作用**：这是前端项目的「通信兵」，负责与后端服务器通信。

**主要功能**：
- 发送请求获取数据
- 处理响应数据
- 统一管理API接口

### 5. `backend/src/main.ts` - 后端应用的入口文件

**作用**：这是后端项目的「大脑」，负责初始化整个后端应用。

**主要功能**：
- 导入并配置NestJS实例
- 设置中间件和全局过滤器
- 启动HTTP服务器

### 6. `backend/src/modules/auth/auth.service.ts` - 认证服务

**作用**：处理用户登录、注册等认证相关功能。

**主要功能**：
- 用户登录验证
- JWT令牌生成
- 用户信息管理

## 8️⃣ API接口文档

后端提供了完整的API接口，可通过以下地址访问：

**Swagger文档**：http://localhost:3001/api/docs

### 主要接口

#### 认证接口
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/register` - 用户注册
- `GET /api/auth/me` - 获取当前用户信息
- `POST /api/auth/refresh` - 刷新Token
- `POST /api/auth/logout` - 退出登录

#### 学生管理接口
- `GET /api/students` - 获取学生列表
- `POST /api/students` - 创建学生
- `GET /api/students/:id` - 获取学生详情
- `PATCH /api/students/:id` - 更新学生信息
- `DELETE /api/students/:id` - 删除学生

#### 教师管理接口
- `GET /api/teachers` - 获取教师列表
- `POST /api/teachers` - 创建教师
- `GET /api/teachers/:id` - 获取教师详情
- `PATCH /api/teachers/:id` - 更新教师信息
- `DELETE /api/teachers/:id` - 删除教师
