## 修复的所有问题汇总

### 1. ✅ 后端依赖注入问题
**问题**：使用`type`导入导致NestJS无法注入依赖
**修复**：将所有Service、Repository、Controller的`type`导入改为正常导入
**影响文件**：15个后端文件

### 2. ✅ ValidationPipe配置问题
**问题**：`forbidNonWhitelisted: true`导致请求被拒绝
**修复**：移除该配置，添加`enableImplicitConversion`
**影响文件**：`backend/src/main.ts`

### 3. ✅ 密码哈希问题
**问题**：种子数据中的密码哈希不正确
**修复**：使用bcrypt动态生成正确的哈希
**影响文件**：`backend/prisma/seed.ts`

### 4. ✅ 查询参数类型转换问题
**问题**：查询参数是字符串，Prisma期望数字
**修复**：在Repository中添加`Number()`强制转换
**影响文件**：`student.repository.ts`, `teacher.repository.ts`

### 5. ✅ 异常过滤器错误代码问题
**问题**：所有错误都返回`code: "0000"`
**修复**：根据HTTP状态码返回不同的错误代码
**影响文件**：`backend/src/common/filters/http-exception.filter.ts`

### 6. ✅ JWT认证守卫未全局应用
**问题**：`/auth/me`接口返回500，`req.user`为undefined
**修复**：在AppModule中添加全局JWT守卫
**影响文件**：`backend/src/app.module.ts`, `backend/src/common/guards/jwt-auth.guard.ts`

### 7. ✅ 角色格式不匹配问题
**问题**：后端返回`SUPER_ADMIN`，前端期望`R_SUPER`
**修复**：在AuthService中添加角色映射
**影响文件**：`backend/src/modules/auth/auth.service.ts`

### 8. ✅ 环境变量配置问题
**问题**：`VITE_STATIC_SUPER_ROLE=SUPER_ADMIN`与后端返回的`R_SUPER`不匹配
**修复**：修改为`VITE_STATIC_SUPER_ROLE=R_SUPER`
**影响文件**：`.env`

### 9. ✅ Token赋值错误（最后一个问题）
**问题**：`token.value = token`尝试在字符串上创建属性
**修复**：将局部变量重命名为`accessToken`避免冲突
**影响文件**：`src/store/modules/auth/index.ts`

## 技术要点总结

### 1. NestJS依赖注入
构造函数参数不能使用`type`导入：
```typescript
// ❌ 错误
import type { UserService } from './user.service';

// ✅ 正确
import { UserService } from './user.service';
```

### 2. 角色映射
后端需要将数据库角色转换为前端期望的格式：
```typescript
const roleMapping: Record<string, string> = {
  'SUPER_ADMIN': 'R_SUPER',
  'ADMIN': 'R_ADMIN',
  'TEACHER': 'R_TEACHER',
  'STUDENT': 'R_STUDENT'
};
```

### 3. 环境变量配置
前端环境变量必须与后端返回的角色格式一致：
```env
VITE_STATIC_SUPER_ROLE=R_SUPER
```

### 4. Ref变量命名
避免局部变量与ref变量同名：
```typescript
// ❌ 错误
const token = ref('');
function test() {
  const token = 'value';
  token.value = token; // 错误！
}

// ✅ 正确
const token = ref('');
function test() {
  const accessToken = 'value';
  token.value = accessToken;
}
```
